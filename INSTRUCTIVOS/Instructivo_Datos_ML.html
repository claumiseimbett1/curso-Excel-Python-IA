2. Qu es NumPy?

NumPy (Numerical Python) es una biblioteca fundamental de Python para computacin cientfica y anlisis numrico.

Caractersticas principales:

  1. Arrays multidimensionales: Estructura de datos principal llamada ndarray (N-dimensional array) que permite trabajar con matrices nxm (n filas x m columnas) y vectores (1xm vector fila, o nx1 vector columna) de manera eficiente
  2. Operaciones matemticas: Funciones optimizadas para operaciones matemticas, algebraicas y estadsticas
  3. Rendimiento: Implementado en C, es mucho ms rpido que las listas de Python para operaciones numricas
  4. Base para otras libreras: Pandas, scikit-learn y muchas otras libreras de ciencia de datos estn construidas sobre NumPy

Conceptos clave:

  • Array (np.array): Estructura de datos principal, similar a una lista pero ms eficiente. Un array es un arreglo donde se guardan los datos numricos
  • Operaciones vectorizadas: Permite realizar operaciones en arrays completos sin bucles, por ejemplo multiplicacin de filas por columnas como en lgebra lineal, o operacin punto a punto entre vectores o matrices
  • Broadcasting: Permite operaciones entre arrays de diferentes tamaos
  • Indexacin avanzada: Acceso eficiente a elementos usando ndices, mscaras booleanas, etc.

Ejemplo tpico de uso:

import numpy as np

# Crear un array
numeros = np.array([1, 2, 3, 4, 5])

# Operaciones matemticas
print(numeros * 2)  # Multiplicar todos los elementos por 2
print(numeros.mean())  # Calcular la media
print(np.sort(numeros))  # Ordenar el array

En el proyecto, NumPy se usa para operaciones numricas, clculos matemticos y como base para las operaciones de pandas y scikit-learn.

3.2 Qu es Matplotlib?

Matplotlib es una biblioteca de Python para crear visualizaciones y grficos estticos, animados e interactivos.

Caractersticas principales:

  1. Visualizacin de datos: Permite crear grficos de lnea, barras, dispersin, histogramas, etc.
  2. Personalizacin: Control total sobre colores, estilos, etiquetas y formato
  3. Integracin: Funciona bien con NumPy y Pandas
  4. Exportacin: Puede guardar grficos en mltiples formatos (PNG, PDF, SVG, etc.)

Conceptos clave:

  • Figure (fig): El contenedor completo que puede contener uno o ms grficos
  • Axes (ax): Un grfico individual dentro de una figura
  • Plot: La funcin que dibuja los datos en el grfico
  • Subplot: Mltiples grficos organizados en una cuadrcula

Ejemplo tpico de uso:

import matplotlib.pyplot as plt
import numpy as np

# Crear datos
x = np.linspace(0, 10, 100)
y = np.sin(x)

# Crear grfico
plt.plot(x, y)
plt.title('Grfico de Seno')
plt.xlabel('X')
plt.ylabel('Y')
plt.show()

En el proyecto, Matplotlib se usa para visualizar datos, resultados de modelos y crear grficos para presentaciones o reportes.

5.2 Qu es StandardScaler?

StandardScaler es una herramienta de preprocesamiento de scikit-learn que estandariza las caractersticas (features) de tus datos.

Qu hace?

Transforma los datos para que tengan:

  • Media = 0
  • Desviacin estndar = 1

Frmula

Para cada valor x, calcula:

x_estandarizado = (x - media) / desviacin_estndar

Por qu es importante?

  1. Escalas diferentes: Si tus variables tienen rangos muy distintos (ej: edad 0-100, ingresos 0-1000000), algunos algoritmos (regresin, SVM, KNN, redes neuronales) pueden verse sesgados por las variables con valores ms grandes.
  2. Convergencia ms rpida: Ayuda a que algoritmos basados en gradientes converjan ms rpido.
  3. Comparacin justa: Pone todas las variables en la misma escala.

Ejemplo prctico:

# Datos originales
edad: [25, 30, 35, 40, 45]
ingresos: [30000, 50000, 70000, 90000, 110000]

# Despus de StandardScaler
edad: [-1.41, -0.71, 0, 0.71, 1.41]
ingresos: [-1.41, -0.71, 0, 0.71, 1.41]

Ambas variables quedan en la misma escala, facilitando la comparacin.

Uso tpico:

from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)  # Usa la misma media y std del entrenamiento

Importante: Ajusta (fit) solo con los datos de entrenamiento y luego transforma tanto entrenamiento como prueba con esos mismos parmetros para evitar data leakage.

5.5 np.random.seed(42) Explicacin

np.random.seed(42) fija la semilla para la generacin de nmeros aleatorios en NumPy. En anlisis de datos, esto es fundamental para garantizar la reproducibilidad de los experimentos.

Qu hace?

  • Fija el punto de partida del generador de nmeros aleatorios.
  • Con la misma semilla, la secuencia de nmeros aleatorios ser la misma en cada ejecucin.
  • til para reproducibilidad: los resultados son consistentes entre ejecuciones.

Por qu el nmero 42?

Es una convencin (referencia a La gua del autoestopista galctico). Puede ser cualquier entero; lo importante es usar el mismo valor para reproducir resultados. El uso del nmero 42 como parmetro random_state en el aprendizaje automtico es una referencia humorstica a la serie de ciencia ficcin "La gua del autoestopista galctico" de Douglas Adams. En la serie, el nmero 42 es conocido como la "Respuesta a la ltima Pregunta de la Vida, el Universo y Todo", aunque la pregunta real es desconocida. Es una referencia caprichosa que ha sido adoptada por la comunidad de programacin y ciencia de datos.

Importante: No hay una razn tcnica especfica para usar el nmero 42 sobre cualquier otro valor. El propsito de establecer una semilla aleatoria (random_state) en aprendizaje automtico es asegurar la reproducibilidad. Cuando usas la misma semilla aleatoria, obtendrs la misma secuencia de nmeros aleatorios cada vez que ejecutes tu cdigo. Esto es til para depurar, compartir resultados y comparar diferentes modelos.

La comunidad de ciencia de datos suele elegir nmeros comnmente reconocidos como el 42 como semilla aleatoria por tradicin, pero cualquier valor entero funcionar igual de bien para lograr la reproducibilidad

Ejemplo prctico:

import numpy as np

# Sin seed: cada ejecucin da nmeros diferentes
np.random.random(3)  # Puede dar: [0.374, 0.950, 0.732]
np.random.random(3)  # Puede dar: [0.598, 0.156, 0.156]

# Con seed: siempre da los mismos nmeros
np.random.seed(42)
np.random.random(3)  # Siempre da: [0.37454012, 0.95071431, 0.73199394]

np.random.seed(42)  # Resetear al inicio
np.random.random(3)  # De nuevo: [0.37454012, 0.95071431, 0.73199394]

Cundo usarlo?

  • En machine learning y anlisis de datos: para que los experimentos sean reproducibles.
  • En pruebas: para garantizar resultados consistentes.
  • Al compartir cdigo: para que otros obtengan los mismos resultados.

Resumen: Fija el generador para obtener la misma secuencia en cada ejecucin, til para reproducibilidad. El 42 es arbitrario, aunque popular.

4.2 Qu es openpyxl?

openpyxl es una biblioteca de Python para leer y escribir archivos Excel (.xlsx, .xlsm).

Caractersticas principales:

  1. Lectura y escritura: Permite leer datos de archivos Excel existentes y crear nuevos archivos Excel
  2. Formato completo: Soporta lectura y escritura de formatos, estilos, frmulas, grficos, etc.
  3. Manipulacin de hojas: Permite crear, eliminar, copiar y modificar hojas de clculo
  4. Compatibilidad: Funciona con archivos Excel modernos (.xlsx, .xlsm), no con formatos antiguos (.xls)

Conceptos clave:

  • Workbook (wb): Representa todo el archivo Excel (libro de trabajo)
  • Worksheet (ws): Representa una hoja individual dentro del libro
  • Cell: Representa una celda individual con su valor, formato, etc.
  • Notacin: Las filas se numeran desde 1, las columnas pueden ser nmeros (1, 2, 3) o letras ('A', 'B', 'C')

Ejemplo tpico de uso:

from openpyxl import load_workbook

wb.close()

En el proyecto, openpyxl se usa cuando necesitas leer datos directamente desde archivos Excel con ms control que pd.read_excel(), o cuando necesitas crear o modificar archivos Excel con formato especfico.

5. Qu son los modelos PKL (Pickle) y por qu se guardan?

Los archivos .pkl (Pickle) son archivos binarios de Python que permiten serializar (convertir a bytes) y deserializar (recuperar) objetos de Python para guardarlos en disco y reutilizarlos despus. Los bytes son representaciones binarias compuestas de ceros (0) y unos (1).

5.1 Qu es Pickle?

Pickle es un mdulo de Python que permite guardar objetos de Python (como modelos de machine learning entrenados) en archivos binarios (.pkl). Esto es til porque:

  1. Preservacin del estado: Guarda el modelo completo con todos sus parmetros entrenados
  2. Reutilizacin: Permite usar el modelo sin tener que entrenarlo de nuevo
  3. Persistencia: El modelo se guarda permanentemente en disco
  4. Eficiencia: Evita re-entrenar modelos que pueden tardar horas o das

Por qu se guardan los modelos?

Despus de entrenar un modelo de machine learning, normalmente:

  • El entrenamiento puede tomar mucho tiempo (minutos, horas o das)
  • El modelo contiene parmetros aprendidos (pesos, coeficientes, rboles, etc.)
  • Necesitas usar el modelo en el futuro sin volver a entrenarlo
  • Quieres compartir el modelo con otros usuarios o sistemas

Sin guardar el modelo: Tendras que entrenarlo cada vez que lo necesites (muy ineficiente)

Con el modelo guardado: Solo cargas el archivo .pkl y haces predicciones inmediatamente. La idea es que el despliegue de este modelo se realice en Excel, lo cual se aprender a hacer en este curso.

5.2 Qu contiene un modelo PKL?

Un archivo .pkl puede contener:

  1. El modelo entrenado: Con todos sus parmetros y pesos aprendidos

  2. Objetos auxiliares:

    • StandardScaler (escalador de datos)
    • LabelEncoder (codificador de etiquetas)
    • SimpleImputer (para valores faltantes)
    • Lista de nombres de caractersticas (features)
    • Metadatos del modelo

  3. Informacin adicional:

    • Mtricas de evaluacin (R, RMSE, MAE)
    • Parmetros del modelo
    • Fecha de entrenamiento

5.3 Alternativas a Pickle

  • Joblib: Similar a Pickle pero ms eficiente para arrays grandes de NumPy (recomendado para scikit-learn)
  • HDF5: Para modelos grandes y datasets
  • ONNX: Formato estndar para intercambiar modelos entre frameworks
  • TensorFlow SavedModel / PyTorch: Formatos especficos de estos frameworks

5.4 Qu es un Framework?

Un framework (marco de trabajo) es un conjunto de herramientas, bibliotecas y reglas que facilita el desarrollo de software. Proporciona una estructura base y funciones comunes para que te enfoques en la lgica de tu aplicacin.

Analoga simple

Si construir una aplicacin fuera construir una casa:

  • Sin framework: Compilas cada herramienta desde cero (martillo, tornillos, diseo).
  • Con framework: Ya tienes herramientas y un plano base; te enfocas en los detalles.

Caractersticas principales

  1. Estructura predefinida: Define cmo organizar tu cdigo.
  2. Funcionalidades comunes: Incluye funciones listas (autenticacin, manejo de datos, etc.).
  3. Patrones de diseo: Usa buenas prcticas ya probadas.
  4. Ahorro de tiempo: Evita escribir cdigo repetitivo.

Ejemplos comunes de Framework

  • Para Machine Learning: TensorFlow, PyTorch, scikit-learn
  • Para web: Django (Python), Flask (Python), React (JavaScript)
  • Para datos: pandas, NumPy

Framework vs. Librera

  • Librera: Herramientas que eliges usar cuando quieres (ej: pandas).
  • Framework: Estructura que sigue tu cdigo; tu cdigo "llena" el framework. Por ejemplo, en machine learning con scikit-learn la estructura que sigue el cdigo es: fit, predict y finalmente score.

En resumen: un framework da estructura y herramientas para desarrollar ms rpido y con mejores prcticas.

6.1 Qu es PyTorch?

  • Loss Function: Funcin que mide qu tan lejos estn las predicciones de los valores reales

Cundo usar PyTorch:

  • Redes neuronales profundas: Para modelos complejos con muchas capas
  • Deep learning avanzado: CNN, RNN, Transformers, etc.
